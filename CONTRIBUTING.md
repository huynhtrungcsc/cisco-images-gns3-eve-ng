# Contributing Guide

Thank you for your interest in contributing to this repository.  
All contributions — new images, fixes, and improvements — are welcome.

---

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button at the top right of this page and clone your fork locally.

### 2. Add or Update Content

- Add a new image entry to the correct section in `README.md`.
- Follow the existing table format — keep columns consistent: `#`, filename, download link, and type/version.
- Ensure all file names are wrapped in backticks (e.g. `` `c3725-adventerprisek9-mz.124-15.T14.image` ``).

### 3. Verify Your Links

Before submitting, verify that all download links are:

- **Publicly accessible** — no sign-in required
- **Correct** — link directly to the file, not a folder
- **Hosted on a reliable service** — Google Drive, MEGA, OneDrive, etc.

### 4. Submit a Pull Request

- Use a clear, descriptive PR title following the Conventional Commits format:
  - `feat(gns3): add Cisco IOSv 15.9 image`
  - `fix: correct broken download link for c7200`
  - `docs: improve usage notes for EVE-NG`
- In the PR description, briefly explain what was added or changed and why.

---

## Commit Message Convention

This project follows the [Conventional Commits](https://www.conventionalcommits.org/) standard:

```
<type>(<scope>): <short summary>

[optional body]
```

| Type | When to use |
|------|-------------|
| `feat` | Adding a new image or section |
| `fix` | Correcting a broken link or wrong information |
| `docs` | Improving documentation or descriptions |
| `chore` | Maintenance tasks (formatting, cleanup) |

---

## Code of Conduct

- Be respectful and constructive in all interactions.
- Only contribute images intended for **educational and lab purposes**.
- Do not submit links to pirated or illegally distributed commercial software outside its intended use.

---

## Questions?

Open an [Issue](../../issues) and describe your question or suggestion.
